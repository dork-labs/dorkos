State 5 — Error

Connection Failed

Error guides the exit — specific message + resolution path. Token stays editable.

Design Rationale

  • Error message guides the exit — specific reason + link to fix
  • Two actions: "Try again" (primary) + "Change token" (escape hatch)
  • "Change token" opens the token input inline — no navigation
  • Red dot in header (no pulse — stable error state)
  • Toggle stays available so user can try again after fixing
  • Settings still accessible if user needs to adjust config

Component Architecture

Clean decomposition for DX excellence — from 1 file × 570 lines to focused, testable components. 

features/settings/ui/
├── TunnelDialog.tsx           ~120 lines — shell + state machine
├── TunnelLanding.tsx          ~60 lines  — illustration + "Get started" CTA
├── TunnelSetup.tsx            ~80 lines  — token input + format validation
├── TunnelConnected.tsx        ~90 lines  — URL card + copy + QR + latency
├── TunnelConnecting.tsx       ~50 lines  — progress steps animation
├── TunnelError.tsx            ~40 lines  — error card + resolution
├── TunnelSettings.tsx         ~100 lines — collapsible settings panel
└── TunnelOnboarding.tsx       ~112 lines — existing illustration (reused)

Total: ~650 lines across 8 files (avg 80 lines each)
