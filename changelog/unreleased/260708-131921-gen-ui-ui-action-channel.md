### Added

- Interactive widgets can now talk back to the agent. Click a widget button, pick from a list action, or submit a widget form, and the agent picks up right where you left off — it receives what you did (the action, any form values, and which widget) and responds in its next turn.
- Buttons show a spinner while the action is sent and surface a clear message if the agent is busy, so it's always obvious whether a click landed.
- Every runtime (Claude Code, Codex, OpenCode) is taught what an interaction looks like, so agents can design widgets that expect a response.
