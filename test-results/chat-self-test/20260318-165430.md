## Issues Found

### Critical Bug: New sessions crash with "Agent stopped unexpectedly" — Bug

**Observed:** Every new message to any session resulted in "Agent stopped unexpectedly. An error occurred during execution." with details "Claude Code process exited with code 1". The chat was completely non-functional.

**Expected:** New sessions should create a fresh Claude Code conversation and stream the response.

**Root cause:** `apps/server/src/services/runtimes/claude-code/claude-code-runtime.ts:252-257`

The `sendMessage()` method auto-creates sessions with `hasStarted: true` unconditionally. This was added in commit `0ee26cb45` to fix resume for sessions that existed before a server restart. However, `hasStarted: true` causes `executeSdkQuery()` to set `sdkOptions.resume = session.sdkSessionId`, and `sdkSessionId` is initialized to the **DorkOS UUID** (not the SDK session ID). The SDK then tries to resume a non-existent session and crashes.

**Fix applied:** Check whether a JSONL transcript file exists before setting `hasStarted: true`. Added `TranscriptReader.hasTranscript()` (lightweight `fs.access` check) and modified `sendMessage()` to use it.

**Files changed:**
- `apps/server/src/services/runtimes/claude-code/transcript-reader.ts` — added `hasTranscript()` method
- `apps/server/src/services/runtimes/claude-code/claude-code-runtime.ts` — use `hasTranscript()` in `sendMessage()`
