TRIGGERS a turn and returns immediately — it does NOT stream tokens (ADR-0264). The turn runs server-side and its events are delivered solely on the durable `GET /api/sessions/{id}/events` stream (the single delivery path). The `202` body carries the CANONICAL session id: for a brand-new session this is the real id assigned during the turn (it differs from the client-supplied id), so the client re-keys its URL and `/events` subscription to it. To avoid missing the turn, a client should be subscribed to `/events` before (or concurrently with) this POST.
