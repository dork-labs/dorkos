## Demographics

- **Age**: 25-45
- **Role**: Product manager / marketing manager / non-technical founder
- **Company**: Any size
- **Technical level**: Beginner — uses ChatGPT/Claude.ai daily, but through web UIs only
- **Tools**: ChatGPT, Claude.ai, Notion, Google Docs, Zapier

## Why They're an Anti-Persona

Jordan wants a prettier ChatGPT. They heard "AI agents" are powerful and want one, but they expect a hosted dashboard, drag-and-drop workflows, natural language configuration, and someone else to run it all. They would never own a system — install it, configure it, learn its shape, recover when it breaks. They want to rent an outcome.

**The disqualifier is operator mentality, not technical skill** _(redrawn 2026-07-09)_. Earlier versions of this persona drew the line at "non-technical," but real usage falsified that: Ikechi (`the-ai-native-founder.md`) and Lil (`the-private-professional.md`) are non-developers who self-host DorkOS precisely because they want ownership and control. What separates them from Jordan is not what they know — it's what they're willing to own. A hospital manager who insists her data stays on her machine is more of a dork, in the brand's sense, than a developer who pastes everything into a consumer chat app. "Dork" was never a credential; it's caring too much.

DorkOS still deliberately does not accommodate Jordan: no hosted no-install offering, no visual workflow builder, no promise that nothing ever needs configuring. Attempting to would compromise the design for everyone who came to own something.

## What Jordan Wants That DorkOS Will Never Provide

- A hosted, no-install web app
- Visual workflow builders instead of config files
- Plain English instead of cron syntax
- No terminal required at any step
- Customer support

## Quote

"I just want to tell the AI what to do and have it happen. Why do I need to install anything?"

## Value of This Anti-Persona

When feature requests arrive for "simpler onboarding" or "no-code configuration", this persona is the filter — but apply the redrawn line. If the request would primarily serve someone who won't operate their own system (hosted, zero-ownership, rented outcomes), it's out of scope. If it removes a jargon wall for an operator who happens not to code — a readable error message, a plain-language doc, a desktop install path — that serves Ikechi and Lil and is in scope. The test is never "is this user technical?" It's "will this user own it?"
