# Demo Video Scripts: Three Brain-Melters

> Positioning review deliverable (July 2026). Three distinct demo concepts for the flagship video. Each is written as a beat sheet with timings, what's on screen, and the caption/voiceover. Some beats are aspirational; every script ends with an honest "what's real today vs not yet" ledger so we never ship a dishonest cut (brand rule: honest by design; a demo that lies is a refund waiting to happen).
>
> Production grammar for all three: real screen capture, no mockups, no stock footage, no music-video editing. Cream UI, monospace captions, silence or one quiet synth pad. Captions carry the story so the silent/X version works; VO is a light overlay for YouTube. The product is the star; the human is the point.

## The shared spine

Every script answers the same three questions in order, because that's the order a skeptical dev asks them:

1. "What am I looking at?" (the cockpit, in the first 5 seconds)
2. "What does it do that my terminal doesn't?" (the melt)
3. "What's the catch?" (self-hosted, MIT, your keys: the trust close)

And every script ends on the same two lines:

> **Intelligence doesn't scale. Coordination does.**
> `curl -fsSL https://dorkos.ai/install | bash`

## Script 2: "One Cockpit, Any Agent" (the vendor-neutrality flex)

**Length:** 75 seconds. **Concept:** the claim no first party can film. Three vendors' agents, one screen, racing and cooperating. The HN/comparison-page weapon. Colder and more technical than Script 1 on purpose.
**Format trick:** split-screen triptych that collapses into one cockpit view.

Quick cuts of alt-tabbing chaos.                                                                                                                                                                                                                                                                           | "You already run more than one agent. They don't know about each other. Neither does anything else."                                                  |
| 0:08-0:16 | The three terminals shrink and slide into a single browser window: the DorkOS fleet screen. All three sessions live, streaming, each tagged with its runtime badge.                                                                                                                                                                                                                                                 | "DorkOS runs them all. Claude Code. Codex. OpenCode. One cockpit."                                                                                    |
| 0:16-0:30 | **The race.** User selects one gnarly bug ticket, clicks "run on all three." Three isolated worktrees spin up; three agents attack the same bug in parallel, streams scrolling side by side. Diff views land one by one; a compare screen shows all three patches with tests-passed badges and cost per attempt (`$0.11`, `$0.38`, `$0.19`). User picks the winner with one click; the losers' worktrees evaporate. | "Same task, three brains, isolated worktrees. Pick the best answer. Throw the rest away. **Reality ledger:** Three runtimes in one cockpit, per-session runtime choice, worktrees, model selectors, session streaming: real. Also real (founder note 2026-07-06): the `private` beat's local-model session, because **OpenCode supports local models** — per-session local routing is shippable now even though the automatic router is not; verify end-to-end before filming (GTM pillar matrix). **Not yet:** run-on-all-three fan-out with a compare screen (workflow exists in pieces; the compare UI doesn't), requirement-based routing and the savings ticker (the flagship aspirational feature, see `02-positioning.md` §4.2), mid-conversation runtime hand-off with context transfer (hard; transcript-to-prompt bridging is plausible v1), automatic outage re-routing (not built). This script is deliberately half roadmap: it's the film version of the category-defining move, and building toward it is the point.

**Why it melts:** nobody has ever seen three frontier vendors' agents race in one UI, and the "sixty-eight cents" compare screen turns an abstract architecture claim (vendor neutrality) into a visceral, greedy, developer-brain moment.

## State-based cuts: match the film to what's actually true (added 2026-07-06)

The three flagship scripts assume beats that may not survive the GTM pillar tests (`09-gtm-plan.md` §2.0/§2.2). Rather than delay filming or fake beats, we maintain a ladder of cuts keyed to product state. Film the cut the product has earned; upgrade the cut as pillars go green.

**Cut 0: "The Cockpit" (45s, filmable the day install + session durability are green).**
Requires only hardening steps 1-2. Beats: `dorkos` → every existing session already there (10s) → live streaming session with tool approval (10s) → three runtime badges on the session list, switch runtime for a new session (10s) → restart the server on camera, everything still there (10s) → trust close + end card (5s). No Tasks, no Telegram, no tunnel: nothing untested. This is the site-hero and README GIF _this month_; the restart-on-camera beat is its quiet flex.

**Cut 1: Script 1 minus unverified beats.** If Telegram passes its smoke test but reply-steering isn't built: film "2:47 AM" with the notification arriving but no reply (drop the 0:24-0:40 reply moment, keep the 7am review). The story still lands; the reply beat joins when the feature ships.

**Cut 2 and 3:** the full Scripts 1-2 as specced above, gated on their ledgers.

### Surface-announcement mini-scripts (the "keep launching" videos)

Each Tier-B/C surface (`09` §2.0) launches later with its own 30-45s film. Same grammar: real capture, captions, the two-line close.

**"The Vault" (Obsidian announcement, 40s).** Beat sheet: an architecture doc open in Obsidian (5s) → the DorkOS pane slides in beside it; user asks "does the auth service already handle token refresh?" without leaving the doc (10s) → the agent answers with a code reference; user keeps writing, flow unbroken (10s) → the same session, already visible in the web cockpit and on the phone (10s) → close: "Your notes were in the room. DorkOS, now inside Obsidian." (5s). _Gate:_ films only after the plugin passes a clean-vault test and one week of real dogfooding in the founder's own vault; this announcement claims flow preservation, so the film must be one continuous unfaked take.

**"The Desktop" (Electron announcement, 30s).** Beat sheet: dock icon click (3s) → the cockpit as a native window, no browser chrome (7s) → a native notification lands while the app is backgrounded; click lands on the exact session (10s) → menu-bar/tray glance showing fleet status (7s) → close: "The control panel, on the desk." (3s). _Gate:_ packaged build installs from a signed artifact on a clean Mac (DOR-155 done); the notification and tray beats film only if those features exist by then; otherwise the cut is dock icon → native window → close, and the announcement copy stays modest.

**"Plugs In" (ecosystem-judo clip, 20s, from `07-website-changes.md` §4.5).** Install a real Claude Code plugin → it appears in Cursor via harness sync → one click schedules its skill nightly. Three captions. _Gate:_ superset verification (`09` §2.5).

**"Offline" (local-model clip, 20s).** Wi-Fi toggled off on camera → an OpenCode session on a local model completes anyway → caption: "Some code never leaves. Now the agent doesn't either." _Gate:_ the OpenCode local-model pillar test.

## Which to make first

**Cut 0 immediately** (it needs only the Week 1-2 hardening), then **Script 1** in whichever cut the pillar tests have earned by Week 5: it's the emotional flagship, and its two honest gaps (Telegram reply steering, nightly cost line) are small features worth shipping anyway. Cut a 30s version (beats 1, 4, 5, end card) for X and the site hero. **Script 2** ships second as the launch-week differentiator once the compare screen exists even in minimal form; its full router act waits for the routing v1. **Script 3** is the conference talk and the /story page centerpiece; film it last, when the marketplace onboarding glue makes act one true and the multi-agent pillar is verified.

One rule across all three: **every artifact shown must be real** (real PRs, real diffs, real costs, real timestamps). Aspirational _features_ get built before their beat gets filmed; we never fake a beat with editing. The reality ledgers above are the build list.
